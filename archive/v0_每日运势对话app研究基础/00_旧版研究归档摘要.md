# 旧版研究归档摘要

日期：2026-05-25

状态：归档。作为 v1「人生信号」轻量 Web App 的底层素材，不再作为当前产品主方案。

## 归档原因

当前策略已经从「每日运势对话 App」调整为「类似人生 K 线的超轻量 Web App 小程序」。

旧版研究仍有价值：
- 计算引擎可复用。
- 数据源可复用。
- 安全边界可复用。
- Peter / Andrew / Bill 的团队机制可复用。

旧版研究不再直接推进：
- 长对话 App。
- 完整内部 wiki。
- 大规模命理知识库。
- 深度技术设计。
- 多体系玄学模块。

## 旧版核心结论

### 产品目标

旧版目标是一个每日运势解惑 App：
- 友善。
- 专业。
- 提供情绪支撑。
- 输出可解释的每日建议。

### 团队机制

Peter：
- 审命理规则。
- 判断哪些结论能说。
- 判断哪些结论必须弱化。

Andrew：
- 找数据源。
- 找开源项目。
- 验证计算引擎。
- 做 benchmark 数据管理。

Bill：
- 把命理逻辑转成产品需求。
- 设计输入、输出、体验和评测。

### 已验证资产

可继续使用：
- `lunar-python`：MVP 八字/历法计算 baseline。
- `sxtwl`：节气时间校验。
- `cnlunar`：黄历字段交叉校验。
- `MingLi-Bench`：内部 benchmark 候选。
- 节假日 JSON：用于日常语气、工作日/休息日判断。

需谨慎使用：
- 古籍文本：需要 Peter 审核。
- 官方比赛题库：复用权利未确认。
- 用户真实内容：不能随意抓取。
- 大师排行榜：不能作为真理来源。

## 转入 v1 的资产

### 直接转入

1. 出生信息输入逻辑。
2. 历法、节气、干支、五行、黄历字段。
3. 计算 trace 必须可回溯。
4. 产品输出不能做确定性恐吓。
5. 产品定位应弱化「算命」，强化「信号、导航、自我理解、行动建议」。

### 暂不转入

1. 长对话能力。
2. 紫微斗数。
3. 周易卦象。
4. 复杂专家咨询。
5. 付费报告体系。
6. 大规模用户画像 memory。

## v1 承接方式

v1 只做一个短链路：

输入少量信息 -> 生成今日/当前人生信号 -> 展示路况、导航或几何图 -> 给一句行动建议 -> 生成分享卡。

旧版研究作为后台规则资产，不在前端暴露复杂命理术语。

## 当前决策

当前产品主线不是「问答」，而是「一张可理解、可截图、可分享、可复访的人生状态图」。
