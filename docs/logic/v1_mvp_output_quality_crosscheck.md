# v1 MVP 输出质量交叉验收

日期：2026-05-25

结论：第一版技术 Demo 不够 C 端；已改为“用户先看结论，再玩路线，再看依据”的版本。

## 用户视角验收标准

- 3 秒内知道今天整体路况。
- 10 秒内知道现在该做什么、下一步做什么、不要做什么。
- 30 秒内能点击 A/B/C 路线，并理解每条路线的收益和代价。
- 命理依据可读，不以 JSON 形式直接暴露给普通用户。
- 所有结论仍可回溯到内部 trace。

## Bill 产品复核

原问题：
- 主结论弱。
- 分数像内部工具。
- 路线切换反馈不明显。
- Trace 不适合普通用户。

已调整：
- 顶部新增“今日路况”主结论。
- 底部新增“现在 / 收益 / 代价”三张行动卡。
- 路线卡展示推荐指数、收益、代价。
- 点击 A/B/C 后行动卡和导航指令同步变化。
- Trace 改为“为什么是这个结果”的用户版依据。

保留问题：
- 分享卡仍是轻交互，未生成真实图片。
- 路线时间仍是隐喻，需要用户测试是否误解。

## Peter 命理复核

通过点：
- 用户页不直接堆复杂术语。
- 不输出确定性命运结论。
- 命理解释使用“参考”“节奏”“风险”语言。
- 保留干支、节气、宜忌、五行关系作为依据层。

保留问题：
- 五行权重仍是产品验证权重。
- 宜忌到行动建议的映射需要规则卡审核。
- 冲煞是否进入红色风险段仍需 Peter 最终确认。

## Andrew 数据复核

通过点：
- 继续使用 `lunar-javascript@1.7.7`。
- 生成链路仍是 deterministic。
- 用户版依据不替代技术 trace。
- 技术 trace 仍可在依据面板中展开。

保留问题：
- 生产版仍需多引擎 parity test。
- 真太阳时未实现。
- 位置/IP 还未进入计算。

## 当前输出质量判断

可以进入本地用户试玩。

不能进入正式灰度。

下一步优先级：
1. 收集 5-10 个真实用户问题。
2. 看用户是否理解“人生路况”的路线隐喻。
3. 用 Peter 审核后的规则卡替换当前权重。
4. 再做分享卡和每日回访机制。
