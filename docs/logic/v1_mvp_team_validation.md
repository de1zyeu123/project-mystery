# v1 MVP 团队确认记录

日期：2026-05-25

结论：可以进入本地 MVP 测试。

## Peter 确认

通过条件：
- 使用历法、节气、干支、五行、黄历作为内部特征是合理方向。
- 用户结果页不直接展示复杂术语。
- 不输出确定性命运判断。
- 不输出医疗、法律、投资、关系操控建议。
- 出生时间缺失时必须降低精度。

保留意见：
- 五行到六维评分的具体权重仍需 Peter 审核。
- 黄历宜忌进入行动建议的边界仍需细化。
- 冲煞能否影响红色风险段，需要再做规则卡。

## Andrew 确认

通过条件：
- MVP 使用 `lunar-javascript@1.7.7`，可在浏览器端计算。
- 当前可以输出干支、节气、黄历宜忌、冲煞等结构化特征。
- 所有结果保留 trace，便于复现。
- 渲染层不直接依赖命理计算。

保留意见：
- 生产版仍需与 `lunar-python` / `sxtwl` / `cnlunar` 做 parity test。
- 立春等节气边界需要单独 golden cases。
- 当前 MVP 不做真太阳时。

## Bill 确认

通过条件：
- 已完成输入页到结果页闭环。
- 结果以导航图呈现。
- A/B/C 路线可切换。
- 结果页有六维评分、路线建议、交互详情和 trace。
- 用户可在本地测试完整流程。

保留意见：
- 规则权重先用于产品验证，不作为最终命理标准。
- 下一阶段需要 30 个 golden cases。
- 需要验证用户是否理解 `min` 是路线强弱隐喻，不是真实时间。
