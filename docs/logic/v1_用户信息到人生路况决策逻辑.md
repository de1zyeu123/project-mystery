# v1 用户信息到人生路况决策逻辑

日期：2026-05-25

状态：工作版，供产品、规则、研发共同评审。

## 1. 核心结论

人生路况不应该由大模型自由生成。

正确链路是：

用户信息 -> 特征层 -> 规则层 -> 评分层 -> 路线决策树 -> Route JSON -> SVG 地图渲染。

大模型只做两件事：
- 把用户自由问题归类。
- 在固定结论内做文案润色。

大模型不决定：
- 哪条路红。
- 哪条路绿。
- 是否推荐变道。
- 是否加速。
- A/B/C 路线耗时。
- 风险等级。

## 2. 产品原则

### 2.1 用户感知

用户看到的是：
- 像导航一样的结果。
- 现在在哪。
- 下一步往哪走。
- 要不要变道。
- 要不要加速。
- 哪条路线更稳。

用户不需要看到：
- 八字术语。
- 五行推导。
- 黄历字段。
- 评分公式。
- 模型 prompt。

### 2.2 系统原则

系统必须做到：
- 相同输入在同一天结果稳定。
- 不同用户有明显差异。
- 页面风格统一。
- 结论可解释。
- 风险表达不恐吓。
- 所有结果能回溯到规则。

## 3. 输入设计

### 3.1 必填输入

| 字段 | 用途 | 是否敏感 | 产品说明 |
|---|---|---:|---|
| 出生日期 | 生成基础命理特征 | 中 | 最低门槛 |
| 当前关注点 | 决定问题场景权重 | 低 | 让结果更像用户当前问题 |

### 3.2 推荐输入

| 字段 | 用途 | 是否敏感 | 产品说明 |
|---|---|---:|---|
| 出生时间 | 提升八字精度 | 中 | 可选，不知道也能测 |
| 出生地 | 时区/真太阳时/地域代际语境 | 中 | 推荐手填 |
| 性别 | 只在涉及大运顺逆等传统逻辑时使用 | 中 | v1 可选 |

### 3.3 当前上下文输入

| 字段 | 来源 | 用途 | 默认策略 |
|---|---|---|---|
| 当前城市 | 用户手选或 IP 粗定位 | 时区、节假日、空间语境 | 优先手选 |
| 当前日期时间 | 系统时间 | 今日干支、节气、日内建议 | 自动 |
| IP 粗定位 | IP geolocation | 城市级兜底 | 只用城市，不存精确位置 |
| 浏览器语言/地区 | 浏览器 | 默认语言、时区兜底 | 只做体验优化 |

### 3.4 用户问题输入

用户可以自由问，也可以点问题。

推荐首屏问题按钮：
- 今天适合推进吗？
- 今天要不要变道？
- 今天适合联系 TA 吗？
- 今天适合谈钱吗？
- 今天适合做决定吗？
- 这周我该稳一点还是冲一点？

这些问题的价值：
- 比「事业/财富/关系」更有趣。
- 直接映射到导航语言。
- 帮用户更高效地表达真实需求。

## 4. 输入分层

每个用户请求被拆成四层。

### 4.1 身份层 ProfileVector

稳定信息。

```json
{
  "birthDate": "1996-04-18",
  "birthTime": "08:30",
  "birthplace": "Hangzhou",
  "gender": "female",
  "calendarType": "solar",
  "birthTimeConfidence": "known"
}
```

用途：
- 生成基础命理盘。
- 决定长期底色。
- 决定路线形态偏好。

### 4.2 当下层 ContextVector

当前状态。

```json
{
  "currentDate": "2026-05-25",
  "currentTime": "12:45",
  "currentCity": "Shanghai",
  "timezone": "Asia/Shanghai",
  "isHoliday": false,
  "locationSource": "manual"
}
```

用途：
- 生成今日信号。
- 决定日内节点。
- 决定「20:00 后加速」这类时间标注。

### 4.3 问题层 IntentVector

用户想问什么。

```json
{
  "rawQuestion": "今天要不要联系 TA？",
  "intent": "relationship_contact",
  "domain": "relationship",
  "decisionType": "contact_or_wait",
  "riskSensitivity": "medium"
}
```

用途：
- 调整维度权重。
- 改变路线命名和底部指令。
- 决定哪些支路显示。

### 4.4 趣味层 FlavorVector

增强差异感，但不能改变核心结论。

```json
{
  "tone": "gentle_direct",
  "metaphorPack": "navigation",
  "seed": "user_123|2026-05-25|v1"
}
```

用途：
- 决定文案轻重。
- 决定部分标签表达。
- 决定非核心视觉细节。

## 5. 确定性与随机性边界

### 5.1 必须确定的部分

这些不能随机：
- 公历/农历转换。
- 节气。
- 干支。
- 五行基础分布。
- 黄历字段。
- 节假日。
- 安全拦截。
- 核心分数区间。
- 推荐路线。
- 风险等级。

### 5.2 可以有可控随机的部分

这些可以变化，但必须用 seed 固定：
- 非核心提示语。
- 地图支路轻微形状。
- 标签的同义表达。
- 分享卡副标题。
- 趣味问题排序。

随机方式：

```text
seed = user_id_or_session_id + date + product_version
```

效果：
- 同一个用户当天看到稳定结果。
- 第二天可以自然变化。
- A/B test 可以复现。

### 5.3 不能随机的部分

不能为了好玩随机：
- 红色风险段。
- 是否建议联系某人。
- 是否建议消费/投资。
- 是否建议重大决定。
- 安全拒答。

## 6. 特征计算层

Andrew 负责。

### 6.1 命理特征

| 特征 | 来源 | 用途 |
|---|---|---|
| 年/月/日/时柱 | `lunar-python` | 基础盘 |
| 当前日干支 | `lunar-python` | 今日信号 |
| 当前节气 | `lunar-python` + `sxtwl` 校验 | 季节/阶段感 |
| 五行分布 | 干支映射 | 维度底色 |
| 生肖 | 年支 | 轻量用户理解 |
| 黄历宜忌 | `cnlunar` 交叉校验 | 行动建议 |
| 冲煞 | 黄历字段 | 风险提示 |

### 6.2 位置特征

位置不直接决定命运。

位置只做三件事：
- 时区校正。
- 节假日/工作日语境。
- 当前环境表达。

可用字段：

| 字段 | 用途 |
|---|---|
| 当前城市 | 显示城市语境 |
| 时区 | 当前日期与出生时间校正 |
| 工作日/节假日 | 行动建议语气 |
| 天气 API | v2 可选，只做表达增强 |

v1 不使用：
- 精确经纬度。
- 用户实时移动轨迹。
- 小区/街道级定位。
- 位置历史。

### 6.3 问题特征

问题分类枚举：

| intent | 示例问题 | 主要维度 |
|---|---|---|
| `career_push` | 今天适合推进工作吗？ | 推进力、支持度、风险 |
| `career_change` | 要不要换方向？ | 稳定性、风险、变道 |
| `relationship_contact` | 要不要联系 TA？ | 关系顺畅、情绪、风险 |
| `money_commitment` | 今天适合谈钱吗？ | 财富安全、稳定性、风险 |
| `mood_support` | 今天怎么让自己舒服一点？ | 情绪、稳定性、休整 |
| `general_today` | 今天整体怎么样？ | 六维均衡 |

自由问题先分类到 intent。

无法分类时，回到 `general_today`。

## 7. 评分层

Bill 负责产品映射，Peter 审核含义。

内部六维：

```json
{
  "momentum": 72,
  "stability": 48,
  "support": 64,
  "relationship": 42,
  "wealthSafety": 58,
  "risk": 67
}
```

### 7.1 维度定义

| 维度 | 产品含义 | 用户语言 |
|---|---|---|
| momentum | 推进意愿与适行动程度 | 适不适合往前走 |
| stability | 容错率与稳态 | 稳不稳 |
| support | 贵人/资源/外部助力 | 有没有外部支持 |
| relationship | 沟通与关系阻力 | 关系路顺不顺 |
| wealthSafety | 资源承诺安全度 | 谈钱/消费稳不稳 |
| risk | 冲突、误判、情绪波动 | 风险段多不多 |

### 7.2 分数来源

每个分数由三部分组成：

```text
score = base_profile_score * 35%
      + daily_signal_score * 45%
      + intent_context_adjustment * 20%
```

解释：
- `base_profile_score`：出生信息带来的长期底色。
- `daily_signal_score`：今日干支、节气、黄历等。
- `intent_context_adjustment`：用户问事业/关系/财富导致的权重变化。

v1 不需要把公式做得复杂，重点是可追溯。

## 8. 路况决策树

### 8.1 总体路况

```text
if risk >= 75 and stability < 45:
  roadState = "congested"
elif momentum >= 70 and stability >= 55:
  roadState = "smooth"
elif momentum >= 60 and risk >= 60:
  roadState = "fast_with_construction"
elif momentum < 45 and support < 50:
  roadState = "rest_area"
else:
  roadState = "slow_climb"
```

### 8.2 是否变道

```text
if intent is career_change and support >= 70 and risk < 55:
  laneChange = "consider_later"
elif risk >= 60 or stability < 50:
  laneChange = "not_recommended"
else:
  laneChange = "small_adjustment"
```

用户语言：
- `not_recommended` -> 暂不建议变道。
- `small_adjustment` -> 可以小幅调整。
- `consider_later` -> 可以观察后再变道。

### 8.3 是否加速

```text
if momentum >= 70 and risk < 50:
  speedAdvice = "accelerate_now"
elif momentum >= 60 and risk < 70:
  speedAdvice = "accelerate_later"
elif risk >= 70:
  speedAdvice = "slow_down"
else:
  speedAdvice = "keep_speed"
```

用户语言：
- `accelerate_now` -> 可以小步加速。
- `accelerate_later` -> 晚些时候再加速。
- `slow_down` -> 低速通过。
- `keep_speed` -> 保持当前节奏。

### 8.4 推荐路线

```text
if risk >= 70:
  recommendedRoute = "A"
elif momentum < 45:
  recommendedRoute = "C"
elif momentum >= 78 and stability >= 60:
  recommendedRoute = "B"
else:
  recommendedRoute = "A"
```

解释：
- A 稳妥路是默认推荐。
- B 快速路只在推进力和稳定性都高时推荐。
- C 绕行路只在低能量或情绪支持场景推荐。

### 8.5 隧道/出口/红段

```text
if risk >= 60:
  addSegment("red_risk")

if stability < 55 and support >= 55:
  addSegment("tunnel_near_exit")

if support >= 60:
  addNode("next_exit")

if intent in ["relationship_contact", "money_commitment"] and risk >= 60:
  addAnnotation("do_not_force")
```

## 10. IP 与当前位置如何使用

### 10.1 原则

位置是增强因子，不是命运因子。

用户应该能理解：
- 位置影响当前环境。
- 位置不决定你的人生。

### 10.2 优先级

```text
用户手选当前城市 > 浏览器授权定位 > IP 粗定位 > 默认城市
```

### 10.3 可用场景

| 场景 | 用法 |
|---|---|
| 时区 | 计算当前日期和日内节点 |
| 节假日 | 调整工作/休整建议 |
| 城市语境 | 北京/上海/杭州等只影响表达 |
| 天气 | v2 可选，增强「人生天气」表现 |
| 方位 | v2 可选，必须由 Peter 审核规则 |

### 10.4 禁用场景

不使用：
- 精确定位追踪。
- 历史轨迹。
- 住址推断。
- 附近商户推断。
- 根据位置做敏感画像。

### 10.5 产品呈现

如果使用 IP 粗定位：

```text
当前城市：上海（根据网络粗略判断，可修改）
```

如果用户手选：

```text
当前城市：上海
```

## 11. 趣味问题系统

### 11.1 目标

让用户更容易问出真实问题。

不是只给分类：
- 事业。
- 财富。
- 关系。

而是给具体问题：
- 今天适合推进吗？
- 该不该变道？
- 要不要联系 TA？
- 今天能不能谈钱？
- 现在该冲还是该稳？

### 11.2 问题池结构

```json
{
  "questionId": "career_push_001",
  "display": "今天适合推进吗？",
  "intent": "career_push",
  "riskLevel": "low",
  "routeBias": "A",
  "resultFocus": ["momentum", "risk", "support"]
}
```

### 11.3 问题推荐逻辑

问题排序可以个性化：

```text
if user is first_visit:
  show general + career + relationship
elif prior_focus exists:
  show prior_focus related questions
elif current context is workday_morning:
  show career_push questions
elif current context is weekend_evening:
  show relationship/mood questions
```

问题排序可以用可控随机。

问题结果不能随机。

## 12. 大模型使用边界

### 12.1 可以用

| 场景 | 用法 |
|---|---|
| 自由问题分类 | 把用户问题映射到 intent |
| 文案润色 | 在固定结论内改写 |
| 解释生成 | 给一句温和解释 |
| 分享卡文案 | 生成更顺口的标题 |

### 12.2 不能用

| 场景 | 原因 |
|---|---|
| 直接生成 UI | 不可控 |
| 直接决定路线 | 难以解释 |
| 直接生成红/绿路段 | 风险不可控 |
| 直接做命理判断 | 规则漂移 |
| 直接回答高风险问题 | 安全不可控 |

### 12.3 推荐 Prompt 形态

模型输入只能是结构化结论：

```json
{
  "intent": "career_push",
  "selectedRoute": "A",
  "roadState": "slow_climb",
  "laneChange": "not_recommended",
  "speedAdvice": "accelerate_later",
  "forbiddenClaims": ["一定", "注定", "必然", "大凶"]
}
```

模型输出只能是：

```json
{
  "softSummary": "今天适合稳中推进，不适合突然改变节奏。",
  "shareTitle": "我今天走 A 稳妥路",
  "supportLine": "慢一点不是停下，是在等视野变清楚。"
}
```

## 13. Mapbox / OpenAI 的定位

### 13.1 Mapbox

Mapbox 可以用于真实地图版本。

适合：
- 真实城市底图。
- 真实当前位置。
- 真实路线层。
- GeoJSON 叠加。
- 点击路线、点击节点、弹窗说明。

不建议作为 v1 默认：
- 我们当前是语义地图，不是真实地理导航。
- 路线 A/B/C 是人生状态映射，不是经纬度路线。
- Mapbox 需要 token、计费和服务条款评估。
- 真实地图会增加定位隐私和用户误解。

推荐架构：

```text
Route JSON -> SvgSemanticMapRenderer
Route JSON -> MapboxGeoRenderer
```

也就是说，产品逻辑不绑定 Mapbox。

### 13.2 OpenAI 图像生成

OpenAI 图像生成可以用于设计阶段：
- 生成 UI 概念图。
- 探索地图视觉风格。
- 生成分享卡背景。

不建议用于用户每次生成结果页：
- 成本高。
- 延迟高。
- 风格不可控。
- 交互不可控。

最终产品页应该是代码渲染：
- SVG 可以点击。
- 路线可以切换。
- 节点可以展开。
- 分享卡可以生成。
- 风格由 design token 控制。

## 16. 当前推荐决策

v1 的核心不是「预测一生」。

v1 的核心是：

把用户的出生信息、当前上下文和当前问题，压缩成一张可解释、可点击、可分享的导航图。

推荐产品表达：

```text
你今天不需要知道命运是什么。
你只需要知道：现在在哪，下一步走哪条路，什么时候该慢，什么时候能小步加速。
```
