# V4 人生路况故事拓扑

## 设计原则

- 底图先服务故事，再服务算法。
- 节点只落在陆地、平台、山坡或码头边，不落水。
- 桥、梯子、台阶是地形语义，不是装饰。
- 用户侧不显示节点编号；编号只用于内部校对和算法映射。
- 草图里有两个 `10`，系统内部拆为 `10A` 和 `10B`。

## 拓扑

```text
0 -> 1 / 2
1 -> 3 / 4
3 -> 6 / 7
6 -> 10A
7 -> 11
4 -> 7 / 8
8 -> 11
2 -> 4 / 5
5 -> 8 / 9
8 -> 11
9 -> 10B
```

终点：

- `10A`：山地挑战终点。结果好坏取决于命盘是否扛得住压力。
- `11`：灯塔平稳终点。重秩序、长期积累、稳定成局。
- `10B`：崖边捷径终点。重风险控制，兑现与回撤并存。

## 地形元素

| 路段 | 地形 | 含义 |
|---|---|---|
| `0 -> 1` | 桥 | 走捷径，跨出常规路，进入森林 |
| `0 -> 2` | 平原路 | 走普通道路，先稳住基本盘 |
| `1 -> 4` | 桥 | 把非常规机会接回现实主线 |
| `3 -> 7` | 小桥 | 从森林冒险回到可执行的平原 |
| `3 -> 6` | 台阶/山坡 | 持续挑战，练硬本事 |
| `8 -> 11` | 梯子 | 补门槛后上高地 |
| `5 -> 8` | 回头路/平路 | 下路挑战过高时回到坦途 |
| `5 -> 9` | 梯子 | 崖边险路，需要工具和保护 |

## 命理映射

| 路型 | 命理语义 | 适合说法 |
|---|---|---|
| 森林捷径 | 偏门、食伤、偏财、驿马 | 有机会试新路，但先小步试 |
| 平原主路 | 正官、正印、土气、财库 | 慢一点，把确定的事做扎实 |
| 桥 | 贵人、平台、合作、规则承接 | 需要连接资源，不要孤走 |
| 梯子 | 资质、证据、背书、硬门槛 | 先补条件，再上高地 |
| 山坡 | 官杀压力、杀印相生 | 压力能转成果，但不能硬扛 |
| 崖边路 | 偏财、劫财、波动、风险 | 可以试，但要控制下注 |

## 路线措辞

| 路段 | 名称 | 一句话 |
|---|---|---|
| `0 -> 1` | 走捷径 | 你可以快一步，但要先跨出去。适合试新机会，不适合一次押满。 |
| `0 -> 2` | 走平原 | 这条路慢一点，但更稳。适合先把确定的事做扎实。 |
| `1 -> 3` | 深入森林 | 继续往新路里走，会更有发现，也会更不确定。 |
| `1 -> 4` | 过桥回主路 | 把新机会接回现实。先找平台、合作方或明确规则。 |
| `3 -> 6` | 爬小山 | 会累，但这是练硬本事的一段。适合补技能、拿资格、扛责任。 |
| `6 -> 10A` | 翻山收束 | 挑战到顶，结果会很明确。扛住了就是高位收获，扛不住就是消耗。 |
| `3 -> 7` | 过桥入平原 | 从冒险回到可执行。适合把灵感变成计划。 |
| `7 -> 11` | 平路到灯塔 | 这条路不刺激，但结果清楚。适合长期积累后稳定成局。 |
| `4 -> 7` | 沿主路慢走 | 路长一点，但阻力小。适合稳住主线，逐步放大。 |
| `4 -> 8` | 抄近路上高台 | 可以快一点，但要补一道硬条件。没有梯子就上不去。 |
| `8 -> 11` | 爬梯到灯塔 | 先把该补的条件补齐，再上高地。过了这一层，路会变稳。 |
| `2 -> 4` | 走向主路 | 先回到主线，把眼前确定的事走顺。 |
| `2 -> 5` | 贴崖试路 | 这条路更快，也更窄。适合小步试，不适合重押。 |
| `5 -> 8` | 回到坦途 | 如果硬爬的条件不够，先退回平路。能回头，是为了保住后面的机会。 |
| `5 -> 9` | 爬梯过坎 | 前面不是平路，要靠工具上去。先补资源、方法和保护。 |
| `9 -> 10B` | 梯后坦途 | 过了最险的一段，后面会平一些。结果看你前面有没有守住边界。 |

## LLM 输出约束

每条路线输出四层：

1. `标题`：直接说选择，例如“走捷径”“走平原”。
2. `短句`：20-36 字，说当前处境和适宜动作。
3. `宜/忌`：各 1-2 个现实动作。
4. `详解`：解释命理逻辑，但必须落到行为。

禁用：

- 不说“必然发财”“一定失败”。
- 不把内部编号暴露给用户。
- 不把“偏财、官杀、食伤”裸露成玄学术语堆砌。
- 不把桥、梯子解释成装饰；它们必须对应资源连接或门槛。
