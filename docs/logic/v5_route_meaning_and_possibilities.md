# V5 路线含义与结果可能性

日期：2026-05-31

目标：先定义每条路意味着什么、通向什么可能性，再画底图。

本文是地图重画的上游依据。底图必须服务这张表。

## 1. 总体故事线

用户从 `0` 出发，第一步只有两种选择：

| 起点选择 | 路线 | 含义 |
|---|---|---|
| `0 -> 1` | 走捷径 | 先跨桥进入森林，代表非常规机会、快一步、但不确定性更高 |
| `0 -> 2` | 走平原 | 先走普通道路，代表常规路径、慢一点、但更稳 |

之后路线逐步展开。

最终不是一个答案，而是三类可能：

| 终点 | 结果类型 | 含义 |
|---|---|---|
| `10` | 山地挑战结果 | 通过持续挑战后到达。好坏取决于是否扛得住压力 |
| `11` | 灯塔稳定结果 | 通过平稳积累、规则承接、补足条件后到达 |
| `12` | 崖边捷径结果 | 通过风险路线后到达。可能兑现，也可能回撤 |

## 2. 每条路的定义

| 路段 | 名称 | 地形 | 意味着什么 | 下一步可能 |
|---|---|---|---|---|
| `0 -> 1` | 走捷径 | 桥，入森林 | 用户选择非常规机会。速度更快，但不确定性更高 | 到 `1` 后选 `1 -> 3` 或 `1 -> 4` |
| `0 -> 2` | 走平原 | 平原主路 | 用户选择普通路径。节奏较慢，但更稳 | 到 `2` 后选 `2 -> 4` 或 `2 -> 5` |
| `1 -> 3` | 深入森林 | 森林路 | 继续探索新机会。发现更多，但风险和信息不完整 | 到 `3` 后选 `3 -> 6` 或 `3 -> 7` |
| `1 -> 4` | 过桥回主路 | 桥 | 把非常规机会接回现实主线。需要平台、合作、规则 | 到 `4` 后选 `4 -> 7` 或 `4 -> 8` |
| `3 -> 6` | 爬小山 | 山坡/台阶 | 继续挑战，练硬本事。压力更大，但可能换来更高结果 | 到 `6` 后走向 `10` |
| `6 -> 10` | 翻山收束 | 山地终点 | 挑战到顶。扛住就是高位收获，扛不住就是消耗 | 到 `10` 结束 |
| `3 -> 7` | 过桥入平原 | 小桥 | 从冒险回到可执行路径。把灵感变成计划 | 到 `7` 后走向 `11` |
| `7 -> 11` | 平路到灯塔 | 平路 | 长期积累、稳定成局。结果清楚但不刺激 | 到 `11` 结束 |
| `4 -> 7` | 沿主路慢走 | 中央平路 | 稳住主线，慢一点走。阻力小，适合放大基本盘 | 到 `7` 后走向 `11` |
| `4 -> 8` | 抄近路上高台 | 快路/门槛 | 可以提速，但这里只是通向高台前的快路，不放梯子 | 到 `8` 后走向 `11` |
| `8 -> 11` | 爬梯到灯塔 | 梯子/高台 | 补资质、证据、背书、合作后上高地 | 到 `11` 结束 |
| `2 -> 4` | 走向主路 | 平原会合 | 继续常规路线，先把眼前确定的事走顺 | 到 `4` 后选 `4 -> 7` 或 `4 -> 8` |
| `2 -> 5` | 贴崖试路 | 崖边小路 | 从普通路线中转向更快、更窄的机会。适合小步试，不适合重押 | 到 `5` 后选 `5 -> 8` 或 `5 -> 9` |
| `5 -> 8` | 回到坦途 | 回头路/上行路 | 发现下路挑战过高后，回收到可执行路径。不是失败，是止损与转稳 | 到 `8` 后走向 `11` |
| `5 -> 9` | 爬梯过坎 | 崖边梯子 | 继续走风险路径，但必须靠工具、资源、保护过坎 | 到 `9` 后走向 `12` |
| `9 -> 12` | 梯后坦途 | 下路终点 | 过了最险的一段。结果取决于前面有没有守住边界 | 到 `12` 结束 |

## 3. 关键节点含义

| 节点 | 角色 | 含义 |
|---|---|---|
| `0` | 起点 | 当前人生位置。还没有选择未来路 |
| `1` | 捷径后的第一分岔 | 已进入非常规机会，需要决定继续冒险还是回到主线 |
| `2` | 平原后的第一分岔 | 已选择稳路，需要决定继续主路还是尝试崖边捷径 |
| `3` | 森林深处 | 冒险加深。可以继续挑战，也可以回到可执行平原 |
| `4` | 中央主路节点 | 主线承接点。可以慢走，也可以提速上高台 |
| `5` | 崖边风险节点 | 下路风险点。可以回到坦途，也可以继续爬梯过坎 |
| `6` | 山坡挑战节点 | 持续压力段，通向山地挑战结果 |
| `7` | 中央平稳节点 | 稳定主线，通向灯塔 |
| `8` | 右中高台节点 | 门槛/条件节点，补条件后通向灯塔 |
| `9` | 下路过坎节点 | 风险路线后段，通向右下结果 |
| `10` | 上左终点 | 山地挑战结果 |
| `11` | 上右终点 | 灯塔稳定结果 |
| `12` | 右下终点 | 崖边捷径结果 |

## 4. 结果路径分组

### 4.1 通向 `10`：山地挑战结果

```text
0 -> 1 -> 3 -> 6 -> 10
```

含义：

用户选择捷径后继续深入森林，并愿意承受更高挑战。最终结果取决于抗压能力、资源承接和持续投入。

地图表达：

左侧森林更密，越往上越靠近山地。`3 -> 6` 必须是明显上坡或台阶。`6 -> 10` 是山顶/山脚终点。

### 4.2 通向 `11`：灯塔稳定结果

路径有多条：

```text
0 -> 1 -> 3 -> 7 -> 11
0 -> 1 -> 4 -> 7 -> 11
0 -> 1 -> 4 -> 8 -> 11
0 -> 2 -> 4 -> 7 -> 11
0 -> 2 -> 4 -> 8 -> 11
0 -> 2 -> 5 -> 8 -> 11
```

含义：

无论从捷径、平原还是崖边回来，只要进入 `7` 或 `8`，最终都走向稳定结果。区别在于：`7` 是慢走积累，`8` 是补条件后上高台。

地图表达：

`11` 是右上灯塔。`7 -> 11` 是长平路。`8 -> 11` 是梯子/台阶上高地。`5 -> 8` 必须表达“从风险路回收到稳定路径”。

### 4.3 通向 `12`：崖边捷径结果

```text
0 -> 2 -> 5 -> 9 -> 12
```

含义：

用户先走平原，但中途转向崖边捷径。它可以更快，但必须通过梯子/工具过坎。最终结果同时有兑现和回撤的可能。

地图表达：

`2 -> 5` 是下方风险小路。`5 -> 9` 必须和 `5 -> 8` 分开。`9 -> 12` 是右下终点，不应和灯塔路线混在一起。

## 5. 地图重画硬约束

1. `5` 必须是明显分岔点。
2. `5 -> 8` 是回到稳定路径，不是通往右下终点。
3. `5 -> 9` 是继续风险路径，不是回到高台。
4. `8` 必须连向 `11`，代表补条件后上高地。
5. `9` 必须连向 `12`，代表风险路径后段收束。
6. `10`、`11`、`12` 是三类结果，不能画成同一个终点。
7. 桥只用于连接：`0 -> 1`、`1 -> 3`、`1 -> 4`、`3 -> 7`。
8. 梯子/台阶重点用于：`8 -> 11`、`5 -> 9`、`3 -> 6`。
9. 正式底图不显示编号、文字、黄色线或蓝色标签。

## 6. 明确禁止的旧地图连通

以下连通来自旧底图或 AI 自动补路，必须删除：

| 禁止连通 | 处理 |
|---|---|
| `10 -> 7` 或 `7 -> 10` | 不允许上左山地终点和中央 `7` 直接相连。`10` 只能从 `6` 到达 |
| `1 -> 2` 或 `2 -> 1` | 不允许左侧捷径节点和下方平原节点直接相连。`1` 应该连向 `3` 和 `4` |
| `3 -> 4` 或 `4 -> 3` | 不允许森林节点 `3` 直接连中央节点 `4`。`3` 应该连向 `6` 和 `7` |
| `9` 的第二出口 | `9` 只有一个出口，通向 `12`。不能再画额外岔路、桥或终点 |

红色新增线对应：

| 新增/强调连通 | 处理 |
|---|---|
| `1 -> 4` | 必须是一条清晰桥路，把捷径接回主线 |
| `3 -> 7` | 必须是一条清晰小桥路，从森林回到中央平原 |

蓝色删除标记对应：

- 删除旧图里的 `10 / 7` 联通。
- 删除旧图里的 `1 / 2` 联通。
- 删除旧图里的 `3 / 4` 联通。
- 删除右下 `9` 节点旁多画的额外出口。
