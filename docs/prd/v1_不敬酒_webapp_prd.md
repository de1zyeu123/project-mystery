# v1 PRD：不敬酒 Web App

日期：2026-05-31

状态：工作版，供用户审核。

不包含：
- 技术架构。
- 数据库设计。
- API 设计。
- 复杂命理断语。

## 1. 产品定位

一句话：

用户输入出生年月日、出生时间和出生地，系统生成一个「不敬酒人格水母」和一组拒酒文案。

产品形态：
- 超轻量 Web App / 小程序。
- 单次体验 30-60 秒。
- 结果页是一张人格卡。
- 主视觉是 NPI 风格的不敬酒水母头像。
- 核心传播点是 16 个不敬酒人格，按男女形象扩展为 32 张人格卡。

产品表达：
- 不做酒桌地图。
- 不做长篇命理报告。
- 不判断真实酒量。
- 不鼓励饮酒。
- 核心表达是「你在酒桌上如何守住边界」。

## 2. 目标用户

核心用户：
- 讨厌被劝酒的人。
- 想知道自己在饭局里是什么反应模式的人。
- 喜欢 MBTI、星座、八字人格化内容的人。
- 愿意分享「我是 XX 型不敬酒人格」的人。

非目标用户：
- 未成年人。
- 需要医学戒酒建议的人。
- 需要法律、职场维权判断的人。
- 需要严肃命理咨询的人。

## 3. MVP 目标

v1 只验证五件事：

1. 用户是否愿意输入出生信息。
2. 用户是否被 16 型人格名和 32 张人格卡吸引。
3. 用户是否觉得水母形象好笑、像自己。
4. 用户是否愿意保存或分享结果卡。
5. 用户是否觉得拒酒文案能直接用。

## 4. 核心体验

用户路径：

输入出生信息 -> 生成四维倾向 -> 命中 16 型人格 -> 生成水母形象 -> 生成不敬酒文案 -> 生成分享卡。

首屏输入：

| 字段 | 必填 | 说明 |
|---|---|---|
| 出生日期 | 是 | 公历 / 农历需明确 |
| 出生时间 | 建议必填 | 可选择「不知道」 |
| 出生地 | 建议必填 | 用于时区和节气校验 |
| 性别 | 建议必填 | 只影响形象呈现和文案口吻，不改变核心人格 |

结果页结构：

1. 人格名。
2. 不敬酒水母形象。
3. 一句话判词。
4. 酒桌第一反应。
5. 最容易失守的瞬间。
6. 反抗金句。
7. 拒酒策略。
8. 分享卡。

## 5. 主线映射

### 5.1 总流程

```text
生辰信息
-> 时间标准化
-> 八字基础特征
-> 四个酒桌人格维度
-> 16 型不敬酒人格
-> 性别化形象变体
-> 水母形象
-> 拒酒文案
-> 分享卡
```

### 5.2 四个维度

用 2 x 2 x 2 x 2 生成 16 型。

| 维度 | A 面 | B 面 | 用户可见含义 |
|---|---|---|---|
| 冲突姿态 | 硬 H | 软 S | 遇到劝酒是硬挡，还是软滑过去 |
| 反应时机 | 预 Y | 熬 A | 是开局就设边界，还是熬到临界点才处理 |
| 社交资源 | 借 J | 单 D | 是借人群、规则、同盟挡酒，还是自己扛 |
| 结局动作 | 控 K | 跑 P | 是把局面控回来，还是快速撤离 |

正式展示时不露出字母编码，只展示中文人格名。

### 5.3 八字到维度的规则方向

v1 不做强命理结论，只做规则化倾向映射。

| 命理特征 | 影响维度 | 产品解释 |
|---|---|---|
| 日主强弱 | 冲突姿态 | 判断边界感强弱 |
| 食伤强弱 | 反应时机、结局动作 | 判断表达欲、话术能力、临场反击 |
| 官杀压力 | 冲突姿态、反应时机 | 判断对权威和饭局压力的反应 |
| 印星强弱 | 社交资源 | 判断是否倾向用规则、健康、原则做挡板 |
| 比劫强弱 | 社交资源 | 判断是否借朋友、同事、同盟解围 |
| 财星强弱 | 结局动作 | 判断是否在意面子、人情和交换感 |
| 五行偏性 | 四维微调 | 木主动、火外放、土稳定、金边界、水迂回 |

未知出生时间时：
- 不输出强结论。
- 只给基础人格。
- 结果页显示「精度：基础」。

### 5.4 16 型到 32 张卡

规则：
- 16 型是核心人格。
- 男女是视觉和文案变体。
- 同一人格的男女版必须一眼看出是同一物种、同一性格。
- 男女版不做刻板职业差异，只做发型、表情、姿态、道具使用方式差异。
- 分享卡展示人格名时可以加前缀：`男版 / 女版`，但不改变人格名。

示例：
- `HYJK` 男版：公章触须更像审计员，动作更硬。
- `HYJK` 女版：公章触须更像冷脸 HR，眼神更稳。
- 两者都叫「酒桌纪检委水母」。

## 7. 水母形象输出规则

每个结果必须生成一只固定人格水母。

视觉目标：
- NPI 风格水母头像。
- 人格识别度优先。
- 夸张程度优先。
- 不追求真实水母结构。

输出字段：
- `persona_name`：人格名。
- `persona_code`：内部编码。
- `gender_variant`：男版 / 女版。
- `avatar_prompt`：水母形象生成提示。
- `headline`：一句话判词。
- `first_reaction`：酒桌第一反应。
- `danger_moment`：最容易失守的瞬间。
- `signature_line`：反抗金句。
- `strategy`：拒酒策略。
- `share_copy`：分享卡文案。

形象规则：
- 头小，情绪大。
- 触须要承担性格表达。
- 头发、触须、表情、道具都可以夸张。
- 每个水母都要有一个反差点。
- 视觉要像人格头像，不像医学或生物图鉴。
- 男女版要像同一人格的两个皮肤，不像两个新角色。

示例形象提示：

```text
一只「酒桌纪检委水母」。
小脑袋，严肃表情，透明公章帽。
触须像会议纪要和红色警戒线。
旁边有一只被盖章拒收的酒杯。
整体夸张、好笑、适合社交媒体头像。
```

### 8.1 方向 A：塌感软体派

参考机制：
- Gudetama 的「不想营业」。
- Pusheen 的「低压力可爱」。
- MBTI 人格卡的「一个动作代表一个性格」。

关键词：
- 软。
- 懒。
- 摊。
- 融化。
- 被迫营业。

角色特征：
- 圆软小脑袋。
- 身体像快要摊开的果冻。
- 触须短、软、拖地。
- 表情克制，但情绪写在姿势里。
- 酒杯靠近时，水母不是反抗，是物理性融化。

适合人格：
- `含泪误饮水母`。
- `体面蒸发水母`。
- `提前装醉水母`。
- `端茶登基水母`。

优势：
- 最可爱。
- 最容易做表情包。
- 攻击性低，适合大众传播。

风险：
- 可能不够疯。
- 强人格识别需要靠道具和动作补足。

### 8.2 方向 B：胖圆贴纸派

参考机制：
- Pusheen 的圆形轮廓。
- 贴纸角色的重复识别。
- 每个角色只改少量特征，整体家族感强。

关键词：
- 胖。
- 圆。
- 干净。
- 贴纸。
- 一眼识别。

角色特征：
- 所有水母都是胖圆轮廓。
- 五官极简。
- 颜色温和。
- 触须变成小短手。
- 道具非常明确：公章、茶杯、电话、WiFi、退货标签。

适合人格：
- `酒桌纪检委水母`。
- `杯口封印师水母`。
- `微笑退货水母`。
- `话题烟雾弹水母`。

优势：
- 最统一。
- 最适合 32 张卡批量生产。
- 最适合做头像、贴纸、表情包。

风险：
- 夸张程度需要靠动作和文案加强。
- 如果线条太软，会显得像儿童 IP。

### 8.3 方向 C：低多边形人格派

参考机制：
- MBTI 人物的低多边形、马赛克感。
- 每组人格有统一色系。
- 姿态、道具、剪影直接表达性格。

关键词：
- 切面。
- 几何。
- 角色阵列。
- 32 张成套。
- 人格测试感。

角色特征：
- 水母不是圆软形，而是低多边形果冻。
- 头部有明显几何切面。
- 触须像折纸丝带。
- 每个维度有稳定视觉规则：
  - 硬 H：尖角更多。
  - 软 S：圆角更多。
  - 预 Y：姿态前倾。
  - 熬 A：姿态后缩。
  - 借 J：触须连接外部道具。
  - 单 D：触须收回身体。
  - 控 K：道具在手里。
  - 跑 P：身体朝出口漂。

适合人格：
- 全部 32 张人格卡。
- 尤其适合做一张总览图，像 MBTI 16 型海报。

优势：
- 最像人格测试。
- 最系统。
- 最适合做 16 x 男女的矩阵展示。

风险：
- 可爱度不如前两套。
- 如果做太硬，会丢失水母的软体喜感。

### 8.4 推荐测试方式

先用同一人格做三套样张。

测试人格：
- `酒桌纪检委水母`：测试硬边界。
- `含泪误饮水母`：测试反差和可爱度。
- `提前装醉水母`：测试夸张和表情包感。

每套先出：
- 男版 1 张。
- 女版 1 张。
- 分享卡 1 张。
- 16 型总览小样 1 张。

优先判断：
1. 哪套最想分享。
2. 哪套一眼看懂人格。
3. 哪套最适合批量扩成 32 张。
4. 哪套最不容易像低龄儿童 IP。

## 9. 文案标准

用户可见文案：
- 直接。
- 好笑。
- 夸张。
- 有反差。
- 有画面感。
- 不恐吓。
- 不鼓励拼酒。

推荐句式：
- 「你不是不会敬酒，你是不想把边界交出去。」
- 「你的礼数很足，但酒精没有席位。」
- 「你最危险的不是第一杯，是第三轮开始有人起哄。」
- 「你的策略不是硬刚全桌，而是让这一轮自然断掉。」
- 「你看起来在圆场，其实已经把杯子从剧情里删掉了。」

禁用句式：
- 「你命里必须喝。」
- 「你天生酒量好。」
- 「多喝能转运。」
- 「不喝会得罪贵人。」
- 「硬喝一下就过去了。」

## 10. MVP 范围

Included：
- 出生信息输入。
- 标准化时间处理。
- 四维人格评分。
- 16 型人格结果。

- AI 长篇对话。

## 11. 成功指标

产品指标：
- 输入完成率。
- 结果页停留时长。
- 分享率。
- 保存率。
- 话术复制率。

质量指标：
- 用户反馈「像我」比例。
- 用户反馈「好笑」比例。
- 用户反馈「想分享」比例。
- 低俗 / 劝酒 / 恐吓文案命中率。
- 未成年人拦截成功率。

## 12. 待确认问题

1. 产品名是否固定为「不敬酒」。
2. 16 型人格名是否继续往更荒诞、更疯、更社交媒体化方向推。
3. 水母形象是否统一做成头像卡，还是做成人格海报。
4. 是否需要每个类型配 3 句可复制拒酒话术。
5. 分享卡主视觉是「大头像」还是「头像 + 金句」。
6. 三套视觉方向优先测试哪一套。
