# v1 PRD：人生信号 Web App

日期：2026-05-25

状态：工作版，供用户审核。

不包含：
- 技术架构。
- 数据库设计。
- API 设计。
- 部署方案。

## 1. 产品定位

一句话：

用户输入少量信息，系统生成今日人生路况、天气和状态图，帮助用户理解当前状态并获得行动建议。

产品形态：
- 超轻量 Web App / 小程序。
- 单次体验 30-60 秒。
- 结果页可截图、可分享。
- 后续可扩展为每日复访。

产品表达：
- 不叫「AI 算命」。
- 不叫「八字批命」。
- 暂定表达为「人生信号」「人生路况」「命运导航」。

## 2. 目标用户

核心用户：
- 想快速看今天状态的人。
- 对玄学、星座、MBTI、塔罗、八字有兴趣的人。
- 想获得情绪支撑和行动建议的人。
- 愿意截图分享「我是 XX 型 / 今天路况 XX」的人。

非目标用户：
- 需要严肃命理咨询的人。
- 需要医疗、法律、投资决策的人。
- 需要完整排盘工具的人。

## 3. MVP 目标

v1 只验证四件事：

1. 用户是否愿意完成输入。
2. 用户是否看得懂结果。
3. 用户是否觉得结果「像自己」。
4. 用户是否愿意截图分享。

## 4. 最小输入项

### 4.1 极速模式

用于降低门槛。

| 字段 | 必填 | 说明 |
|---|---|---|
| 出生日期 | 是 | 年/月/日 |
| 当前关注点 | 是 | 事业、财富、关系、情绪、综合 |
| 当前城市 | 否 | 可手选；不默认强制定位 |

特点：
- 适合首次体验。
- 不要求出生时间。
- 只能输出轻量结论。
- 结果页必须显示「精度：基础」。

### 4.2 标准模式

用于生成更可信结果。

| 字段 | 必填 | 说明 |
|---|---|---|
| 出生日期 | 是 | 公历/农历需明确 |
| 出生时间 | 建议必填 | 可选择「不知道」 |
| 出生地 | 建议必填 | 用于时区和真太阳时校验 |
| 当前城市 | 否 | 用于空间语境和节假日 |
| 性别 | 可选 | 只在需要大运顺逆时使用 |
| 当前关注点 | 是 | 控制结果页重点 |

特点：
- 支撑八字基础盘。
- 支撑更完整的路况/导航。
- 可给出更高置信度。

### 4.3 校准模式

不进入首屏。

| 字段 | 必填 | 说明 |
|---|---|---|
| 过去一个低谷年份 | 否 | 用于校准叙事 |
| 过去一个转折年份 | 否 | 用于校准叙事 |
| 过去一个高光年份 | 否 | 用于校准叙事 |

用途：
- 提升用户参与感。
- 提高结果可信感。
- 后续做年度报告或长周期图。

## 5. 输入到结果的支撑逻辑

### 5.1 总流程

用户输入 -> 时间标准化 -> 命理基础特征 -> 今日时间特征 -> 场景权重 -> 信号评分 -> 展示映射 -> 文案生成。

### 5.2 时间标准化

Andrew 负责。

处理：
- 公历/农历转换。
- 出生地时区。
- 出生时间未知标记。
- 节气边界。
- 当前日期、当前节气、当前干支。

底层依据：
- `lunar-python` 做 baseline。
- `sxtwl` 校验节气时间。
- `cnlunar` 校验黄历字段。

规则：
- 出生时间未知时，不输出强个人八字判断。
- 节气边界附近，结果页要降低置信度。
- 不能用日期级结果替代精确出生盘。

### 5.3 基础命理特征

Peter 负责审核规则。

最小特征：
- 年柱、月柱、日柱、时柱。
- 五行分布。
- 日主。
- 生肖。
- 当前日干支。
- 当前节气。
- 黄历宜忌。
- 冲煞。
- 工作日/节假日。

首版不做：
- 紫微斗数。
- 奇门遁甲。
- 周易起卦。
- 复杂格局断语。
- 终身命运定论。

### 5.4 场景权重

Bill 负责产品映射。

用户选择关注点后，系统调整维度权重：

| 关注点 | 权重上调 |
|---|---|
| 事业 | 推进力、贵人/外部支持、风险干扰 |
| 财富 | 稳定性、风险干扰、行动窗口 |
| 关系 | 关系顺畅、情绪能量、沟通窗口 |
| 情绪 | 情绪能量、稳定性、休整建议 |
| 综合 | 六维均衡 |

### 5.5 信号维度

v1 先使用六个内部维度：

| 维度 | 含义 | 用户可见 |
|---|---|---|
| 推进力 | 今天适不适合行动 | 是 |
| 稳定性 | 今天容不容易失误 | 是 |
| 支持度 | 外部帮助和资源信号 | 是 |
| 关系顺畅 | 沟通与互动阻力 | 可选 |
| 财富安全 | 花钱、承诺、资源调度风险 | 可选 |
| 风险干扰 | 冲突、误判、情绪波动 | 是 |

首屏只展示三角形：
- 推进力。
- 稳定性。
- 支持度。

第二屏可展示六边形。

## 6. 结果页方案

### 6.1 页面结构

结果页分五块：

1. 今日一句话结论。
2. 人生天气。
3. 人生路况。
4. 命运导航。
5. 三角形/六边形状态图。
6. 行动建议与分享卡。

### 6.2 第一块：一句话结论

目标：
- 3 秒内给用户判断。

格式：

今天是「小步推进」的一天。整体多云转晴，适合推进旧事，不适合开启新冲突。

字段：
- 今日关键词。
- 整体状态。
- 最适合做什么。
- 最不适合做什么。

### 6.3 第二块：人生天气

目标：
- 给用户一个低压力的实时状态。

展示：
- 天气图标。
- 当前温度式分数。
- 分时段趋势。
- 风险预警。

示例：

今日人生天气：多云转晴。

上午：能见度偏低，适合整理。

下午：气压回升，适合沟通。

晚上：情绪湿度偏高，少做关系判断。

映射规则：

| 内部状态 | 天气表达 |
|---|---|
| 推进力高、风险低 | 晴 |
| 推进力中、风险中 | 多云 |
| 推进力低、风险中 | 阴 |
| 关系/情绪波动高 | 阵雨 |
| 风险干扰高 | 雷阵雨 |
| 稳定性低、信息不清 | 雾 |

### 6.4 第三块：人生路况

目标：
- 让用户知道当前处境和下一步动作。

展示：
- 当前路况。
- 三条支路。
- 下一出口。
- 今日驾驶建议。

示例：

当前路况：爬坡 + 轻微拥堵。

事业主路：可以推进旧项目。

财富支路：不要冲动承诺。

关系支路：适合轻沟通，不适合摊牌。

下一出口：今晚适合复盘。

映射规则：

| 内部状态 | 路况表达 |
|---|---|
| 推进力高、稳定性高 | 顺行 |
| 推进力高、风险高 | 高速但有施工 |
| 推进力中、稳定性低 | 缓行 |
| 推进力低、风险高 | 拥堵 |
| 支持度高、推进力中 | 出现下一出口 |
| 情绪能量低 | 服务区休整 |
| 信息不清 | 雾区慢行 |

### 6.5 第四块：命运导航

目标：
- 把「当前状态」转成「下一步路线」。

v1 处理方式：
- 不单独做复杂地图。
- 作为人生路况下方的导航模块。
- 展示 1 个当前位置、1 个下一出口、1 个风险路段、1 个建议动作。

示例：

当前位置：爬坡路段中段。

下一出口：今晚适合复盘旧计划。

风险路段：不要在情绪低点做关系判断。

导航建议：继续直行，小步推进，不急于变道。

映射规则：

| 内部状态 | 导航表达 |
|---|---|
| 推进力高、稳定性高 | 继续直行 |
| 推进力高、风险高 | 降速通过 |
| 推进力中、支持度高 | 前方出现出口 |
| 推进力低、情绪低 | 进入服务区休整 |
| 风险干扰高 | 建议绕行 |
| 信息不清 | 暂不变道 |
| 关系顺畅低 | 避免急转弯 |

边界：
- 不说「你必须这样做」。
- 不说「错过就没机会」。
- 不把导航建议包装成重大人生决策。

### 6.6 第五块：三角形/六边形

目标：
- 给用户一个可截图的结构化状态。

v1 首屏三角形：
- 推进力。
- 稳定性。
- 支持度。

v1 次屏六边形：
- 事业推进。
- 财富安全。
- 关系顺畅。
- 情绪能量。
- 贵人支持。
- 风险干扰。

展示逻辑：
- 只显示 0-100 分。
- 标出最高项和最低项。
- 配一句解释。
- 不显示复杂命理术语。

示例：

推进力 72，稳定性 48，支持度 64。

结论：适合小步推进，不适合高风险承诺。

### 6.7 第六块：行动建议与分享卡

行动建议格式：

今天适合：
- 推进旧项目。
- 找一个熟人确认信息。
- 把复杂决定延后 24 小时。

今天不适合：
- 冲动承诺。
- 情绪化沟通。
- 高风险消费。

分享卡格式：

我的今日人生路况：隧道末段，多云转晴。

今日关键词：慢进攻。

命运类型：逆周期翻盘型。

## 7. 命运类型系统

v1 可以生成一个轻标签，但不作为主结果。

候选类型：
- 低开高走型。
- 逆周期翻盘型。
- 稳定复利型。
- 贵人驱动型。
- 资源整合型。
- 高波动成长型。
- 晚熟爆发型。
- 孤勇开拓型。

使用规则：
- 标签必须正向。
- 标签不能羞辱用户。
- 标签不能制造宿命感。
- 标签要能被用户截图分享。

## 8. 安全与合规

必须避免：
- 医疗判断。
- 投资交易建议。
- 法律结论。
- 恐吓式命运判断。
- 关系操控建议。
- 「必须」「一定」「注定」。

推荐表达：
- 倾向。
- 更适合。
- 可以优先考虑。
- 今天不必急着决定。
- 把它当成一次自我观察。

产品定位：
- 娱乐。
- 自我探索。
- 情绪陪伴。
- 行动提醒。

## 9. 首版用户路径

1. 用户进入页面。
2. 选择「极速测一下」或「更准一点」。
3. 填出生日期。
4. 选择关注点。
5. 可选填写出生时间和出生地。
6. 点击生成。
7. 看到结果页。
8. 保存或分享卡片。
9. 可选择补充出生时间提升精度。

## 10. 评测指标

### 产品指标

| 指标 | 目标 |
|---|---|
| 输入完成率 | 验证门槛是否足够低 |
| 结果页停留时长 | 验证结果是否值得看 |
| 分享点击率 | 验证传播性 |
| 二次生成率 | 验证趣味性 |
| 次日回访率 | 验证日活潜力 |

### 质量指标

| 指标 | 目标 |
|---|---|
| 计算可追溯率 | 每个结论有输入和规则依据 |
| Peter 审核通过率 | 命理表达不过界 |
| 用户理解率 | 用户能复述结果含义 |
| 负面情绪触发率 | 控制恐吓和焦虑 |
| unsupported claim rate | 控制无依据结论 |

## 11. 人工审核节点

进入原型前，需要用户审核：

1. 是否同意 v1 主形态：人生路况 + 人生天气 + 三角形。
2. 是否同意最小输入：出生日期 + 关注点，出生时间/出生地可选。
3. 是否同意底层逻辑：历法/节气/干支/五行/黄历为基础，紫微/周易暂缓。
4. 是否同意输出边界：不做确定性预测，不做重大决策建议。

## 12. 当前推荐决策

首版不做完整「人生 K 线」。

首版做「人生信号」：
- 第一屏：今日天气。
- 第二屏：当前路况。
- 第三屏：三角形状态。
- 分享卡：命运类型 + 今日关键词。

原因：
- 输入少。
- 结果直观。
- 情绪风险低。
- 可日更。
- 可分享。
- 能复用旧版命理计算资产。
