# Competitor and User Question Inventory

Date: 2026-05-20

## Objective

Collect product patterns for a daily 玄学 dialogue app.

Use competitors for:
- user expectation
- onboarding pattern
- daily retention loop
- trust signals
- output style

Do not use competitors as truth sources.

## User Question Buckets

MVP should support:

| Bucket | Example User Question | Product Need |
|---|---|---|
| Daily fortune | 今天整体运势怎么样？ | fast daily answer |
| Work | 今天适合推进工作吗？ | action guidance |
| Relationship | 今天感情上要注意什么？ | soft, non-manipulative answer |
| Emotion | 我今天很焦虑，怎么理解？ | emotional support |
| Money | 今天财运如何？ | non-investment framing |
| Timing | 今天适合做决定吗？ | risk and timing language |
| Self-understanding | 我这个性格为什么总这样？ | reflective explanation |
| Follow-up | 为什么这么说？ | traceable reasoning |

## Unsafe Question Buckets

These need refusal or soft redirection:

| Bucket | Risk |
|---|---|
| Medical outcome | health harm |
| Legal case outcome | legal advice |
| Financial trading | investment advice |
| Death / disaster | fear and mental harm |
| Pregnancy / children certainty | high sensitivity |
| Relationship control | manipulation |
| Paid ritual cure | scam risk |

## Initial Product Implication

The app should not feel like a 排盘工具 only.

It should feel like:
- daily check-in
- personal reflection
- structured traditional logic
- calm next step

The professional layer should be available, not forced.

Default user output:
1. 今日结论
2. 为什么
3. 建议怎么做
4. 情绪支持
5. 你可以继续问
