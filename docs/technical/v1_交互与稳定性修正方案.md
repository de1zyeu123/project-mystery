# v1 交互与稳定性修正方案

日期：2026-05-25

状态：已进入 preview 修正。

## 1. 问题

当前问题有两个：

1. 地图不够可交互。
2. 规则生成的视觉结果不稳定。

这两个问题不能靠大模型修。

必须靠数据契约、固定模板和渲染器约束。

## 2. 交互方案

地图不是图片。

地图应由可点击对象组成：

| 对象 | 交互 |
|---|---|
| 路线 A/B/C | 点击切换路线 |
| 红色风险段 | 点击展示风险解释 |
| 隧道段 | 点击展示当前阶段解释 |
| 机会出口 | 点击展示下一步建议 |
| 当前位置 | 点击解释“你在这里” |
| 终点旗帜 | 点击解释今日目标 |
| 标注 pill | 点击展示对应说明 |
| 底部路线卡 | 点击切换路线 |

当前 preview 已增加：
- SVG 路线热区。
- SVG 节点热区。
- 键盘可访问。
- 点击后底部 detail panel 更新。

## 3. 稳定性原则

规则引擎不能直接生成坐标。

错误方式：

```text
规则引擎 -> 任意坐标 -> SVG
```

正确方式：

```text
规则引擎 -> 状态枚举 -> 固定路线模板 -> 参数填充 -> SVG
```

## 4. 固定路线模板

v1 只允许使用有限模板。

示例：

| templateId | 场景 |
|---|---|
| `steady_climb` | 缓行爬坡，有风险段，有出口 |
| `fast_risky` | 快速但高风险 |
| `low_pressure_detour` | 绕行休整 |
| `blocked_then_exit` | 先堵后通 |
| `foggy_hold_lane` | 信息不清，保持车道 |

每个模板固定：
- 起点区域。
- 终点区域。
- A/B/C 路线骨架。
- 标签锚点。
- 风险段位置。
- 隧道段位置。
- 出口位置。

规则只能决定：
- 使用哪个模板。
- 哪条路线被推荐。
- 哪些段高亮。
- 标签显示哪种枚举文案。
- 分数和耗时标签。

规则不能决定：
- 任意路线形状。
- 任意标签坐标。
- 任意颜色。
- 任意节点数量。

## 5. 坐标稳定方案

### 5.1 画布坐标

使用固定坐标系：

```text
viewBox = 390 x 572
```

所有模板在这个坐标系内设计。

### 5.2 Anchor 槽位

标签只能放在固定槽位：

```json
{
  "anchors": {
    "current": [42, 494],
    "risk": [160, 328],
    "tunnel": [218, 252],
    "exit": [282, 170],
    "destination": [326, 82]
  }
}
```

### 5.3 标签防重叠

首版不用复杂算法。

使用模板级人工校验：
- 每个模板 3 个屏幕尺寸。
- iPhone SE。
- iPhone 13。
- Desktop phone shell。

后续可增加 label collision pass：
- 如果两个标签距离小于阈值，隐藏次级标签。
- 或把次级标签挪到备用 anchor。

## 6. Route JSON 修正

Route JSON 不应该包含自由坐标。

建议改为：

```json
{
  "templateId": "steady_climb",
  "selectedRoute": "A",
  "states": {
    "roadState": "slow_climb",
    "riskState": "short_congestion",
    "tunnelState": "near_exit",
    "laneChange": "not_recommended",
    "speedAdvice": "accelerate_later"
  },
  "scores": {
    "momentum": 72,
    "stability": 48,
    "support": 64,
    "risk": 67
  },
  "routes": [
    {
      "id": "A",
      "durationLabel": "23 min",
      "tag": "推荐",
      "summary": "小步推进"
    }
  ]
}
```

渲染器负责：
- 找 `steady_climb` 模板。
- 把 states 映射成段、节点和标签。
- 产出最终 SVG。

## 7. 确定性生成

稳定性规则：

```text
same input + same date + same ruleVersion = same Route JSON
```

需要固定：
- `ruleVersion`
- `templateVersion`
- `styleVersion`
- `copyVersion`

需要记录：

```json
{
  "inputHash": "xxx",
  "ruleVersion": "v1.0",
  "templateVersion": "v1.0",
  "styleVersion": "v1.0",
  "copyVersion": "v1.0"
}
```

## 8. 可控随机

可控随机只能影响非核心展示。

允许：
- 分享卡副标题。
- 次级标签同义词。
- 趣味问题排序。

不允许：
- 推荐路线。
- 红色风险段。
- 是否变道。
- 是否加速。
- 节点数量。
- 路线坐标。

## 9. 测试标准

### 9.1 Golden cases

至少 30 个。

每个 case 固定：
- 输入。
- 期望 scores。
- 期望 templateId。
- 期望 selectedRoute。
- 期望 instruction。
- 期望截图 hash 或人工截图。

### 9.2 视觉回归

每个模板跑：
- iPhone SE。
- iPhone 13。
- Desktop phone shell。

检查：
- 无滚动溢出。
- 标签不互相遮挡。
- 路线可点击。
- 节点可点击。
- 底部指令不溢出。

### 9.3 交互回归

必须通过：
- 点击路线 A。
- 点击路线 B。
- 点击路线 C。
- 点击红色风险段。
- 点击隧道段。
- 点击出口。
- 点击当前位置。
